> Этап 1: глубокое исследование (кто уже решал «фото ребёнка → персональный мультик») + выбор AI-стека и план реализации.
> Дата: 2026-06-10. Метод: 6 параллельных агентов-разведчиков (Reddit, ProductHunt, YC, видео-модели, консистентность, indie-builders) + оркестратор-синтезатор. 161 веб-запрос.

## 0. Ключевой вывод одной фразой

Мы **не первые** — Mootion, Pixley (YC F25), Toon Kids, HeroMi уже отгружают «фото → персональный мультик». Но **наша ниша свободна**: быстрый «СДВГ»-монтаж + семья/питомцы/артефакты в кадре + глубокий смысл + русскоязычное исполнение. Главное архитектурное решение — **keyframe-then-animate** (сначала кадр, потом анимация), а консистентность персонажа держим через **zero-training мульти-референс (Nano Banana Pro), НЕ через LoRA**. Самое приятное: наш быстрый монтаж — это не ограничение, а **технический подарок** (быстрые склейки маскируют дрейф персонажа дешевле всего).

## 2. Рекомендованный AI-стек (выбор оркестратора)

**Основной (качество + автоматизация):**
- **Стилизация + библия персонажа + кейфреймы:** Nano Banana Pro (Gemini 3 Pro Image) — лидер 2026 по lock'у личности, ~$0.03–0.04/картинка, ~2 мин. Фолбэк self-host: Flux.1 Kontext / PuLID Flux II.
- **Image-to-video:** Kling (Elements / Character ID) — лучший для 2D/мульт-движения; Runway Gen-4.5 — для кинематографичных геройских кадров. Роутинг по типу кадра.
- **Озвучка:** ElevenLabs multilingual v3 (+ клон голоса родителя). Музыка: Suno / ElevenLabs Music.
- **Липсинк:** OmniHuman 1.5 (заточен под стилизованные/мульт-лица) через fal.ai (~$0.15/сек). Только для говорящих кадров.
- **Сборка:** Remotion / Creatomate + ffmpeg, склейки по бит-сетке музыки, единый колор-грейд.
- **Сценарий/шот-лист:** русскоязычный LLM (GigaChat / YandexGPT), Claude/GPT — для бенчмарка качества при разработке.
- **Оркестрация:** DB-backed пайплайн (n8n или кастомный worker + Postgres), доступ к моделям через fal.ai / Replicate.

**🇷🇺 Слой устойчивости к России (параллельный трек, НЕ основной):** GigaChat/YandexGPT (сценарий) + SaluteSpeech/Yandex SpeechKit (RU-TTS) + Sber Kandinsky 5 (фолбэк видео). Западный стек = лидер качества, отечественный = «всегда оплачиваемая страховка» от блокировки API/платежей. **Строим model-routing абстракцию с первого дня**, чтобы менять модель по кадру.

## 3. Стратегия консистентности персонажа (главный риск)

**Zero-training «золотой референс + пак-лок персонажа»** (НЕ LoRA):
1. Шаг 0 каждого заказа: фото ребёнка → ОДИН канонический мульт-герой на чистом фоне (Nano Banana Pro / Flux Kontext).
2. Из него — пак из 5–6 референсов: фронт нейтральный, профиль, 3/4, крупный план лица, в полный рост, экшн/эмоция.
3. КАЖДЫЙ последующий кейфрейм генерим **с прикреплённым паком** + дословный блок «character DNA» (геометрия, один якорь — напр. красная куртка, одноцветная одежда против «мутации паттернов»).
4. Одобренные кадры переиспользуем как новые референсы — консистентность накапливается.
5. Для видео — подаём виды turnaround в Kling Character ID.

**Почему НЕ LoRA:** каждый клиент = новый персонаж. LoRA = 8–20ч и $30–100 на персонажа — экономически невозможно. LoRA оставить ТОЛЬКО для постоянного каста (рассказчик/маскот-сайдкик) или премиум-апселла «вечный персонаж».

**Финальная страховка — сам монтаж:** клипы 3–5 сек, склейки на движении, единый колор-грейд маскируют остаточный дрейф. Запрошенный «СДВГ»-ритм работает двойную роль: эстетика + инструмент консистентности.

## 4. План реализации (фазы)

### Фаза 1 — Продукт «сценарий + раскадровка» (отгружаем ПЕРВЫМ, без видео)
Валидирует две самых сложных и ценных вещи — эмоциональное качество сценария и консистентность персонажа — при почти нулевой стоимости (COGS ~$1–5, без видео). Все артефакты = прямой вход в Фазу 2 (ноль выброшенной работы).

- [ ] 1.1 Интейк: загрузка фото ребёнка (+ опц. семья/питомцы/вещи), валидация качества лица, контент-безопасность, чистый фронт-кроп. **Хранение детских фото — на своей инфре (PII).**
- [ ] 1.2 Фото → канонический мульт-герой («золотой референс»).
- [ ] 1.3 Библия персонажа: turnaround + эмоции + позы + референсы семьи/питомцев (пак 5–6).
- [ ] 1.4 RU-сценарий с именем/семьёй/питомцами/артефактами и чётким смысловым битом → структурный **shot-list JSON** (ID кадра, длительность ~3с, камера, действие, реплика, правила гардероба/реквизита).
- [ ] 1.5 Кейфрейм-генерация: по 1 полированному стиллу на кадр, с прикреплённым паком (меняем 1 переменную за кадр), с QA до видео-расходов.
- [ ] 1.6 Доставка: красивая раскадровка (PDF/веб-аниматик-галерея) + сценарий + **бесплатное превью с водяным знаком** (механика Toon Kids).
- [ ] 1.7 Модель данных **сохранённой библиотеки персонажа** (примитив Pixley) — заложить уже сейчас → повторные продажи (след. ДР, брат/сестра).

### Фаза 2 — Полный анимированный мультик (позже)
- [ ] 2.1 Image-to-video: shot-list JSON + кейфреймы → Kling (+ Runway для геройских), клипы 3–5с.
- [ ] 2.2 RU-озвучка (ElevenLabs / SaluteSpeech) + опц. клон голоса родителя; сначала строим VO-таймлайн, видео режем под него.
- [ ] 2.3 Музыка (Suno) + извлечение бит-сетки (librosa).
- [ ] 2.4 Липсинк (OmniHuman 1.5) ТОЛЬКО на говорящих кадрах.
- [ ] 2.5 Сборка: склейки ~каждые 3с на движении, привязка к биту, whip-pan/кросс-диссолв (Remotion/ffmpeg).
- [ ] 2.6 Полиш: дефликер/денойз → единый колор-грейд → зерно → апскейл ПОСЛЕДНИМ → мастер + превью.
- [ ] 2.7 Проход consistency-review: регенерация худших ~15–25% кадров.

### Фаза 0 (сквозная, начать сразу) — Инфраструктура
- [ ] 0.1 Model-routing абстракция (западный ↔ отечественный стек, переключение по кадру).
- [ ] 0.2 DB-backed оркестрация: 1 строка на заказ, submit-then-poll для долгих генераций, ретраи, resumability, трекинг стоимости.

## 5. Экономика и цена

COGS готового 3-мин фильма (~40 кадров, хостед-стек): стилизация+библия ~$0.5; кейфреймы с QA ~$3.2; image-to-video ~$12–24; липсинк (~10 говорящих) ~$4–5; TTS+музыка ~$1–3; сборка/апскейл ~$1–2; LLM+оркестрация <$0.5. **ИТОГО ~$22–38, реалистично закладывать ~$30–45** с пере-роллами.

**Следствие:** продаём как **премиум-подарок $25–60+** (якорь — keepsake-книги Magic Story/Hooray Heroes), НЕ конкурируем с шаблонами Toon Kids ($7.99) / HeroMi ($9). COGS Фазы-1 (раскадровка без видео) ~$1–5 — дёшево и быстро валидируется. Бесплатное превью с водяным знаком — мост к премиум-цене.

## 6. Риски

1. **🇷🇺 Доступ к западным API + платежи ненадёжны.** Nano Banana/Kling/Runway/ElevenLabs могут быть заблокированы/неоплачиваемы → отечественный фолбэк (GigaChat/Kandinsky 5/SaluteSpeech) за роутинг-слоем с дня 1.
2. **Юнит-стоимость может раздуться.** 40 кадров × 2–3 варианта → QA-гейтинг до видео-расходов обязателен.
3. **Остаточный дрейф персонажа** даже при 95%+ — один плохой крупный план рушит иллюзию → отдельный consistency-review проход (ручной труд соло-фаундера, главный bottleneck масштабирования).
4. **Детское PII (фото лиц)** — юридический и доверительный риск в РФ → self-host хранения, политика удаления, рассмотреть PuLID Flux II / ComfyUI self-host.
5. **Липсинк — слабейшее звено.** Мульт-рты всё ещё хромают → минимизировать экранный диалог (закадровый голос поверх действия), OmniHuman только на геройских репликах.
6. **«Студийное качество» — подвижная планка** (Mootion/Pixley). Дифференциация держится на быстром монтаже + семья/питомцы + смысл; если исполнение лишь «хорошее» — ценовая война, которую не выиграть.
7. **Соло-исполнение:** пайплайн из 11 стадий. Попытка отгрузить всё разом = провал → строго держать фазовый роадмап (сценарий+раскадровка первыми).
8. **Латентность.** Не позиционировать как «за минуту» (разочарует) → премиум made-to-order подарок (часы, не минуты), даёт место на QA.

---

## 7. Ключевые инсайты (что реально даёт качество)

- **Быстрый «СДВГ»-монтаж = технический подарок:** клипы 3–5с дрейфуют меньше, склейки-на-движении + единый колор-грейд — цитируемый способ №1 маскировать дрейф. Цель «качество» и цель «дёшево» **совпадают**.
- **Экономика запрещает LoRA** → только zero-training мульти-референс (Nano Banana Pro).
- **Keyframe-then-animate** (никогда чистый text-to-video для героя) — несущее решение: per-frame QA до дорогого видео, контроль COGS, отделяет «студию» от «AI-каши».
- **Роадмап сценарий+раскадровка-первыми — правильный**: валидирует сложное и ценное при нулевой видео-стоимости, ноль выброшенных артефактов.
- **Клон голоса родителя** — дешёвый и защитимый ров эмоции, которого нет у прямых видео-конкурентов.
- **Сохранённая библиотека персонажа** (примитив Pixley) — заложить в модель данных Фазы 1 → разовый подарок превращается в повторную выручку.
